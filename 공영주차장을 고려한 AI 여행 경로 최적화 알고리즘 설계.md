# 공영주차장을 고려한 AI 여행 경로 최적화 알고리즘 설계

## 목적

현재 시스템은 TMAP API를 이용하여 차량 이동시간을 최소화하는 경로를 추천한다.

하지만 실제 여행에서는 목적지마다 차량을 이동시키는 것보다, 하나의 공영주차장에 차량을 주차한 뒤 여러 장소를 도보로 방문하는 것이 더 효율적인 경우가 많다.

본 문서는 기존 경로 최적화 알고리즘을 개선하여 **공영주차장을 하나의 거점(Base Node)으로 사용하는 복합 이동 경로(Multi-Modal Route)**를 구현하기 위한 설계 문서이다.

---

# 현재 문제점

현재 계산 방식은 다음과 같다.

```
차량
↓

A

↓

차량

↓

B

↓

차량

↓

C
```

TMAP에서는

```
A→B 차량 5분
B→C 차량 5분
```

으로 계산되므로 총 10분이 소요되는 것으로 판단한다.

하지만 실제 여행에서는

* 주차장 진입
* 빈 주차공간 탐색
* 주차
* 건물 입구까지 이동

과정이 매 목적지마다 반복된다.

예를 들면

```
차량 이동 5분

+

주차 및 건물 접근 7~15분

=

실제 12~20분
```

이 된다.

즉 현재 알고리즘은 실제 체감 이동시간을 과소평가한다.

---

# 개선 목표

차량 이동뿐 아니라

* 주차
* 도보 이동
* 차량 복귀

까지 포함한 실제 이동비용을 계산한다.

---

# 기본 아이디어

반경 내 여러 장소는 하나의 "도보 클러스터"로 묶는다.

예를 들어

```
A
B
C
```

가 서로 300m 이내라면

차량을 각각 이동시키는 대신

```
공영주차장 D

↓

도보

↓

A

↓

B

↓

C

↓

도보

↓

공영주차장 D
```

를 하나의 이동 단위로 계산한다.

---

# 전체 알고리즘

## STEP 1

모든 입력 장소를 좌표화한다.

입력은

* 장소명
* 도로명 주소
* 지번 주소

모두 허용한다.

TMAP 검색 API를 이용하여

```
주소

좌표

POI Category
```

를 획득한다.

---

## STEP 2

장소들을 클러스터링한다.

예)

반경 250~300m 이내 장소들을 하나의 클러스터로 묶는다.

예)

```
Cluster1

A

B

C

Cluster2

E

F
```

---

## STEP 3

각 클러스터 주변 공영주차장을 검색한다.

TMAP POI 검색을 이용하여

반경 내 공영주차장을 찾는다.

공영주차장이 여러 개라면

가장 평균 도보거리가 짧은 주차장을 선택한다.

이를 Base Parking Node로 사용한다.

예)

```
Cluster1

↓

공영주차장 D
```

---

## STEP 4

도보 최적화

공영주차장을 기준으로

```
D

↓

A

↓

B

↓

C

↓

D
```

형태의 도보 루프를 생성한다.

즉

반드시 마지막에는

공영주차장으로 복귀한다.

---

## STEP 5

차량 이동

클러스터 간 이동은

```
현재 위치

↓

D

↓

(도보)

↓

D

↓

다음 공영주차장

↓

(도보)
```

순서로 계산한다.

즉

```
D→A→B→C→D→다음 목적지
```

가 기본 구조가 된다.

---

# 비용 함수(Cost Function)

최적화 대상은 단순 차량 이동시간이 아니다.

다음 식을 최소화한다.

```
총 비용

=

차량 이동시간

+

도보 이동시간

+

주차 이벤트 비용
```

---

# 주차 이벤트(Parking Event)

주차 이벤트는

* 주차장 진입
* 빈 공간 탐색
* 주차
* 건물 입구까지 이동

시간을 포함한다.

---

# 주차 이벤트 계산

모든 장소에 동일한 시간을 적용하지 않는다.

TMAP POI Category를 이용하여 기본시간(Base Time)을 다르게 적용한다.

예)

| Service Category | Base Time |
| ---------------- | --------- |
| 카페               | 4분        |
| 음식점              | 5분        |
| 관광지              | 7분        |
| 문화시설             | 6분        |
| 쇼핑시설             | 9분        |
| 공원               | 3분        |
| 숙박시설             | 5분        |

TMAP의 세부 카테고리는 서비스 카테고리로 변환하여 사용한다.

---

# 혼잡도 보정

사용자가 예상 방문 혼잡도를 선택한다.

기본값은

```
보통
```

이다.

선택지는

* 원활
* 보통
* 혼잡

세 가지이다.

추가시간은

| 혼잡도 | 추가시간 |
| --- | ---- |
| 원활  | +0분  |
| 보통  | +3분  |
| 혼잡  | +7분  |

최종 주차 이벤트는

```
Base Time

+

혼잡도 추가시간
```

으로 계산한다.

예)

쇼핑시설

```
9분

+

혼잡

7분

=

16분
```

---

# POI Category 처리

TMAP에서 반환하는 Category를

우리 서비스 Category로 변환한다.

OpenAI API는 사용하지 않는다.

CSV 또는 JSON Mapping Table을 사용한다.

예)

```
Coffee Shop

↓

카페

↓

4분
```

```
Department Store

↓

쇼핑시설

↓

9분
```

CSV에는

```
TMAP Category

↓

Service Category

↓

Base Time
```

을 저장한다.

---

# 공영주차장 선택 조건

공영주차장을 거치는 것이

기존 차량 이동보다 비용이 낮은 경우에만

공영주차장을 추천한다.

예)

```
기존

A→B→C

28분
```

```
공영주차장

D→A→B→C→D

20분
```

이면

공영주차장을 선택한다.

반대로

도보가 지나치게 길어지는 경우

기존 차량 이동을 유지한다.

---

# 구현 우선순위

## 1차 구현

* 장소 좌표화
* POI Category 수집
* 클러스터 생성
* 공영주차장 검색
* D→A→B→C→D 도보 루프 생성

---

## 2차 구현

* Category Mapping CSV
* Base Time 적용
* 혼잡도 보정
* Cost Function 적용

---

## 3차 구현

* 여러 공영주차장 비교
* 여러 클러스터 간 최적 이동
* 설명 가능한 추천 결과 생성

---

# 최종 목표

기존 내비게이션처럼 단순히 차량 이동시간만 최소화하는 것이 아니라,

* 차량 이동
* 주차
* 도보
* 차량 복귀

를 모두 고려하여 **실제 사용자가 가장 편리하게 이동할 수 있는 여행 동선을 생성하는 AI 경로 최적화 시스템**을 구현한다.
