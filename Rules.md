# 1. Project Vision

ParkRoute AI는 단순한 길찾기 서비스가 아니다.

본 프로젝트는

사용자가 선택한 여행 장소들을

가장 효율적인 여행 일정으로 재구성하는

AI Agent를 개발하는 것을 목표로 한다.

본 프로젝트는

"최단거리"

를 찾는 프로그램이 아니다.

본 프로젝트는

"가장 여행하기 편한 일정"

을 생성하는 프로그램이다.

즉,

주차

도보

차량

예약시간

방문순서

출발지

도착지

를 동시에 고려한다.

==========================================================

# 2. Core Philosophy

본 프로젝트는 다음 철학을 따른다.

Principle 1

사용자가 생각하지 않아도 되는 것을
AI가 대신 생각한다.

----------------------------------------------------------

Principle 2

사용자가 입력한 의도는
항상 존중한다.

AI는

예약시간

바로 다음

다음

등의 조건을 절대로 변경해서는 안 된다.

----------------------------------------------------------

Principle 3

도보는 최소화한다.

사용자는 여행을 즐기러 왔다.

불필요한 도보는 줄인다.

----------------------------------------------------------

Principle 4

주차는 이동의 중심이다.

기존 네비게이션은

출발

↓

도착

만 계산한다.

ParkRoute AI는

주차장을 중심으로

도보 그룹을 생성한다.

----------------------------------------------------------

Principle 5

설명 가능한 AI를 목표로 한다.

최종 결과에는

반드시

추천 이유

가 포함되어야 한다.

==========================================================

# 2. Background

국내 여행을 계획하는 대부분의 사용자는

네이버지도에서

맛집

카페

관광지

숙소

등을 하나씩 검색하여 저장한다.

하지만 저장한 장소들의

방문 순서는

사용자가 직접 결정해야 한다.

이 과정에서 다음과 같은 문제가 발생한다.

• 이동 동선이 꼬인다.

• 불필요하게 차량을 여러 번 이동시킨다.

• 가까운 공영주차장을 찾기 어렵다.

• 도보 이동이 과도하게 많아진다.

• 예약시간을 고려하기 어렵다.

• 식사 후 카페 방문과 같은
사용자 의도를 반영하기 어렵다.

기존 지도 서비스는

A → B

경로 탐색에는 강하지만,

여러 장소를 하나의 여행 일정으로
최적화하는 기능은 부족하다.



==========================================================

# 1. Overview

ParkRoute AI는 단순 최단경로 문제가 아니다.

본 시스템은 다음 요소를 동시에 고려하는 Multi-Constraint Optimization 문제이다:

- 도보 거리 제한
- 차량 이동 시간
- 주차장 이동 구조
- 방문 순서 제약
- 예약 시간
- 사용자 의도 (바로 다음 / 다음)

따라서 본 문제는 다음과 같이 정의된다:

> "Travel Scheduling Optimization Problem with Parking Nodes"

==========================================================

# 2. Hard Constraints (절대 만족 조건)

Hard Constraints는 반드시 만족해야 한다.
위반 시 해당 solution은 폐기된다.

----------------------------------------------------------

HC-001 출발지 고정

설명
사용자가 선택한 출발지는 반드시 첫 노드로 사용된다.

제약
start_node == route[0]

----------------------------------------------------------

HC-002 도착지 고정

설명
사용자가 지정한 도착지는 반드시 마지막 노드로 사용된다.

제약
destination_node == route[-1]

----------------------------------------------------------

HC-003 도보 9분 제한

설명
모든 도보 이동은 9분 이하만 허용된다.

제약
walk_time <= 9 minutes

예외
- 공영주차장 ↔ 장소 이동 포함

결과
9분 초과 edge는 graph에서 제거

----------------------------------------------------------

HC-004 좌표 필수

설명
좌표가 없는 장소는 경로 계산에서 제외된다.

제약
lat != null AND lng != null

----------------------------------------------------------

HC-005 방문 순서 제약 (선후관계)

설명
사용자가 지정한 다음 / 바로 다음 관계는 반드시 유지된다.

예:

A → B (바로 다음)

제약
position(B) = position(A) + 1

A → B (다음)

제약
position(A) < position(B)

----------------------------------------------------------

HC-006 중복 방문 금지

설명
같은 장소는 두 번 방문할 수 없다.

제약
all nodes unique

==========================================================

# 4. Parking Constraint System (핵심 설계)

ParkRoute AI의 핵심 차별점은 "주차 중심 이동"이다.

----------------------------------------------------------

# 4.1 Parking Node 정의

각 지역에는 Parking Node가 존재한다.

Parking Node는 다음 조건을 만족해야 한다:

- 공영주차장 또는 유사 시설
- 차량 접근 가능
- 도보 이동 가능 중심 위치

----------------------------------------------------------

# 4.2 Parking Logic

여행 경로는 다음 구조를 따른다:

[Car Move]
숙소 → 주차장 A

[Walk Group A]
주차장 A → 맛집 → 카페 → 관광지 → 주차장 A

[Car Move]
주차장 A → 주차장 B

[Walk Group B]
주차장 B → 관광지 → 카페 → 주차장 B

----------------------------------------------------------

# 4.3 Parking Cost

각 주차장에는 비용이 존재한다:

- 기본 요금
- 추가 시간당 요금

총 주차 비용:

parking_cost = base_fee + (time_over * unit_fee)

==========================================================

# 5. Time Constraint System

----------------------------------------------------------

# 5.1 Reservation Time Constraint

설명
예약 시간이 존재하는 장소는 반드시 해당 시간 이전/이후 조건을 만족해야 한다.

예:
카페 14:00 예약

제약
arrival_time <= reservation_time

또는
departure_time >= reservation_time

----------------------------------------------------------

# 5.2 Travel Time Model

Travel Time은 TMAP API 기준으로 계산된다.

- car_time
- walk_time

----------------------------------------------------------

# 5.3 Time Feasibility

전체 경로는 시간적으로 가능해야 한다.

arrival_time consistency must be valid.

==========================================================

# 6. Graph Model Definition

Graph = (Nodes, Edges)

----------------------------------------------------------

Nodes

- Place Nodes
- Parking Nodes

----------------------------------------------------------

Edges

- Car edges
- Walk edges

----------------------------------------------------------

Edge filtering rules

- walk_time > 9 min → 제거
- invalid coordinate → 제거

Score =

(1.0 * TotalTravelTime)
+
(0.9 * WalkTime)
+
(0.7 * CarTime)
+
(0.8 * ParkingCount)
+
(0.5 * TotalDistance)

----------------------------------------------------------

Goal

Minimize Score

# 4. Distance Matrix

Matrix 정의:

dist[i][j]

----------------------------------------------------------

Values:

- car_time (minutes)
- walk_time (minutes)
- distance (meters)

----------------------------------------------------------

Rule:

If walk_time > 9:
→ edge removed

==========================================================

# 5. OR-Tools Model Design

# 5.1 Decision Variables

x[i][j] ∈ {0,1}

- i → j 이동 여부

----------------------------------------------------------

u[i]

- 방문 순서 변수 (subtour elimination)

==========================================================

# 5.2 Objective Function

Minimize:

Score =

1.0 * TotalTravelTime
+
0.9 * TotalWalkTime
+
0.7 * TotalCarTime
+
0.8 * ParkingCount
+
0.5 * TotalDistance

==========================================================

# 5.3 Hard Constraints Injection

----------------------------------------------------------

HC-001 Start Node fixed

x[start][*] = 1 only valid start edges

----------------------------------------------------------

HC-002 End Node fixed

all paths must end at destination

----------------------------------------------------------

HC-003 Walk limit

walk_time(i,j) <= 9

----------------------------------------------------------

HC-004 Unique visit

each node visited once

----------------------------------------------------------

HC-005 Order constraint

if A before B:
u[A] < u[B]

==========================================================

# 6. Parking Optimization Strategy

# 6.1 Parking Insertion Logic

Each cluster of nearby places is grouped:

Cluster = Places within walkable radius

For each cluster:

1. Assign nearest parking node
2. Compute walking subgraph
3. Attach car entry/exit

----------------------------------------------------------

# 6.2 Parking Selection Rule

Parking Score:

parking_score =

0.6 * distance_to_first_place
+
0.4 * parking_fee
+
0.2 * congestion_factor

----------------------------------------------------------

Lowest score parking is selected.

==========================================================

# 7. Clustering Strategy

Places are grouped into walking clusters:

Condition:

- walk_time <= 9 min between nodes

Algorithm:

- BFS / Union-Find clustering

Result:

Cluster A:
- Cafe
- Restaurant
- Park

Cluster B:
- Museum
- Shopping mall

==========================================================

# 8. Solver Strategy

# 8.1 Primary Solver

Google OR-Tools Routing Solver

----------------------------------------------------------

# 8.2 Optimization Mode

Mode 1 (Default)
- Minimize walking time

Mode 2
- Minimize total travel time

==========================================================

# 8.3 Search Strategy

- Local search metaheuristics
- Guided descent
- Random restart

==========================================================

# 9. Fallback Strategy

# FC-001 No valid route

→ Increase car usage

----------------------------------------------------------

# FC-002 No parking available

→ Skip cluster grouping

----------------------------------------------------------

# FC-003 Walk constraint too strict

→ Relax walk threshold to 12 min (fallback mode only)

==========================================================

# 10. Complexity Control

Problem size grows exponentially.

Therefore:

- Max nodes: 30
- Max clusters: 6
- Parking candidates per cluster: 3

==========================================================

# 11. Output Format

Solver output:

route = [
    start,
    parking_A,
    place_1,
    place_2,
    parking_A,
    parking_B,
    place_3,
    end
]

This system is NOT:

- shortest path problem

It IS:

- constraint-based human behavior simulation

Cause
- API timeout
- invalid address
- rate limit

----------------------------------------------------------

Response
"좌표 변환 실패"

----------------------------------------------------------

Fallback

----------------------------------------------------------

Cause
일부 장소만 실패

----------------------------------------------------------

Response

- 실패 장소 UI 표시

----------------------------------------------------------

UX Rule

"전체 실패가 아니라 부분 성공 제공"

# 4. Optimization Errors

# ER-006 No Feasible Route

----------------------------------------------------------

Cause
- 도보 9분 제한으로 graph 단절
- 방문 순서 constraint 충돌

----------------------------------------------------------

Response
"조건을 만족하는 경로를 찾을 수 없습니다"

----------------------------------------------------------

1. 도보 제한 9 → 12분 완화 (soft fallback)
2. 주차 클러스터 축소
3. 일부 constraint 무시

----------------------------------------------------------

Priority

Relax constraints in order:

1. Walk constraint
2. Parking constraint
3. Order constraint

==========================================================

----------------------------------------------------------

Cause
- node 수 과다
- constraint complexity 높음

----------------------------------------------------------

Response
"최적화 시간이 초과되었습니다"

----------------------------------------------------------

Fallback

- heuristic solution 사용
- greedy path generation
